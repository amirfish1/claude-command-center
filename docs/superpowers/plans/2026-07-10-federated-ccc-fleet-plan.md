# Federated CCC fleet — staged execution plan

**Spec:** `docs/superpowers/specs/2026-07-10-federated-ccc-fleet-fable-prompt.md`
**Status:** living checklist. Stages are execution order, not optional scope.
**Rule:** small conventional commits, no push. Every stage ends with passing tests.

## Design decisions (locked)

### Identity
- **Node identity:** `~/.claude/command-center/node.json` → `{node_id: uuid4,
  display_name, created_at}`. Created lazily on first read. `node_id` is opaque
  and stable across restarts; display name user-editable via API/UI.
- **Capability manifest:** computed, not stored: `{proto: 1, version,
  engines: [...], features: ["federation", "handoff", "orchestration",
  "group_chat_host", "fleet"]}` plus per-engine `handoff: true|false`
  (claude: true; codex/hermes/others: false until safe migration exists).
- **Repo identity:** canonical Git remote identity `"<host>/<owner>/<repo>"`
  parsed from `origin` URL (https/ssh/git forms normalized, `.git` stripped,
  case-preserving host lowercased). Local-only fallback:
  `"local:<basename>:<first12-of-sha256(root-commit)>"` — stable across
  machines only when the clone shares history; otherwise honest local identity.
- **Global session reference:** `"<node_id>:<native_session_id>"` string form
  plus structured `{node_id, session_id}`. Local refs (bare session IDs) remain
  valid everywhere and mean "this node".

### Peer registry & transport
- **Peer registry:** `~/.claude/command-center/peers.json` →
  `[{node_id, name, transport: {type: "ssh"|"loopback", host?, port?,
  user?, ssh_port?}, secret, added_at, last_seen, version, caps,
  repo_paths: {repo_identity: local_clone_path}}]`. Written only via pairing
  API; secrets never in the repo.
- **Pairing:** explicit two-step: local node generates a pairing request →
  transport probe fetches remote `/api/federation/v1/hello` (identity + caps) →
  user confirms → both sides store the peer with a shared `secret` (32-byte
  urlsafe token generated by the initiating side, delivered over the
  authenticated transport). Every federation mutation carries
  `X-CCC-Peer: <node_id>` + `X-CCC-Peer-Token: <secret>`; the receiving node
  validates against its registry. Unpaired callers get 403 `unpaired_peer`.
- **Transports:**
  - `ssh`: reuse `ssh_multiplexer.BridgeConfig`/`SSHMultiplexer` generalized to
    per-peer instances (not the single env-global). Requests are executed as
    `ssh <dest> python3 -c '<stdin HTTP client>'` hitting the remote CCC on
    **its own loopback** (port read from remote
    `~/.claude/command-center/port.txt` / registry.json). No LAN exposure.
  - `loopback`: direct `urllib` to `127.0.0.1:<port>` — used when the peer is
    another CCC on the same machine (and by the two-node test harness). Still
    inside the loopback trust boundary; pairing token still required.
- **Peer protocol:** versioned under `/api/federation/v1/`:
  - `GET  hello` — node identity card + capabilities (no secret required; safe,
    read-only, used for pairing preflight).
  - `GET  health` — cheap liveness + clock + version (paired only).
  - `GET  sessions` — session inventory (paired only).
  - `GET  repo-inventory?repo=<identity>` — git/worktree state for one repo.
  - `POST route` — envelope `{req_id, hops, action, args}` for spawn / inject /
    ask / group-chat ops executed on the owning node. `hops` decremented per
    forward, max 2; `req_id` deduped in a small persisted ring to make retries
    idempotent.
  - `POST session-import` (staged upload), `POST session-import/commit`,
    `GET  session-export?sid=` — handoff.
- **Staleness:** every aggregated answer carries `observed_at` +
  `stale: true|false` + `source_node`. Peer-fetch failures produce explicit
  `{node_id, error, last_ok_at}` entries, never silent omission.

### Handoff
- **Bundle:** tar stream staged under
  `~/.claude/command-center/federation/staging/<transfer_id>/` containing
  `manifest.json` (version, source/dest node, engine, session_id,
  repo identity, source/dest cwd, branch, commit, files[] with sha256 + bytes,
  created_at) + transcript JSONL + minimal sidecars (name, model choice,
  parent links, provenance). Never repo files, env files, credentials.
- **Path rewrite:** destination project folder = encoded destination cwd
  (Claude's `~/.claude/projects/<encoded>` scheme); transcript lines get `cwd`
  (and other path-bearing metadata fields) rewritten source→dest mapping. Every
  rewrite recorded in `manifest.rewrites[]`; unresolvable absolute paths →
  `manifest.warnings[]`.
- **Import:** validate manifest + hashes in staging, refuse if session exists
  with divergent content (unless explicit `takeover`), then `os.rename` into
  place (atomic per file, transcript last). Failed import leaves destination
  untouched.
- **Ownership lease:** `~/.claude/command-center/federation/leases/<sid>.json`
  `{session_id, owner_node, acquired_at, transfer_id, history[]}` written on
  both sides. Source lease flips to `owner: <dest>` only after destination
  commit ack. Resume/inject on a non-owned session → 409 `not_owner` with the
  owner's node_id. `force=true` (audited) for lost-node recovery.
- **Preflight:** dirty files, unpushed commits, branch/upstream, live processes
  → plan steps; refuses to proceed while dirty (offers "ask owning session to
  commit" via attribution).

### Fleet
- **Inventory:** `/api/fleet/inventory` aggregates local + per-peer
  `repo-inventory` with per-source `observed_at`/`error`. Dimensions kept
  separate: worktree state, unpublished commits (rev-list upstream..HEAD),
  origin default-branch view, PRs (gh, cached), deployments (vercel, cached),
  associated sessions (provenance index).
- **Recommendations:** pure deterministic function inventory → actions, each
  `{id, kind, repo, node, reason, evidence, blockers[], destructive,
  requires[]}`. Gates are objective (clean tree, pushed head, mergeable,
  checks green, reachability proof for worktree removal).
- **Executor:** `/api/fleet/plan` (build) + `/api/fleet/execute` (persisted job
  under `~/.claude/command-center/fleet-jobs/<job_id>.json`, step log, resume
  after restart, precondition revalidation immediately before each mutation).
  Never force-push, never merge red checks, never delete dirty/unproven
  worktrees.
- **Provenance:** `~/.claude/command-center/provenance/<node>.jsonl` appended
  by existing hooks (write events already exist in live-state sidecars) +
  backfill scanner. Attribution scoring hierarchy: hook write event >
  worktree/session ownership > transcript tool path > timestamp correlation;
  emits all candidates + confidence, `unknown` allowed.

## Stages

- [ ] `federation.py` module (stdlib-only): node identity, repo identity

### Stage 2 — federation foundation (server wiring)
- [ ] `/api/federation/v1/hello|health|sessions|repo-inventory|route` in

- [ ] Tests: pairing handshake (loopback, two temp state dirs), unpaired 403,

- [ ] Commit(s): `feat(federation): peer protocol + pairing`,
      `feat(federation): peer setup UI`

## Audit summary (Stage 0, done)

Key reuse points found (file:line refs valid at commit time):

- **State home:** `COMMAND_CENTER_STATE_DIR = ~/.claude/command-center`
  (server.py:60), all `Path.home()`-derived at import. Two isolated instances
  = separate `HOME` + `PORT` + `CCC_EPHEMERAL=1` + `CCC_SKIP_SKILL_INSTALL=1`
  + `CCC_TELEMETRY_DISABLED=1` (pattern proven by
  tests/test_perf_budget.py `big_projects` fixture).
- **Existing identity prior art:** `/api/identity` (server.py:48336) and
  `/api/registry` + `registry.json` (server.py:54674+) — same-machine
  multi-repo peer discovery; same-origin check already loopback-any-port
  (server.py:48382).
- **SSH transport:** `ssh_multiplexer.py` — `BridgeConfig` (env-only, single
  host today), `SSHMultiplexer.run_capture/popen/copy_to_remote` with
  ControlMaster. Federation generalizes to per-peer configs and executes a
  small HTTP client on the remote host targeting the remote CCC's loopback
  port (from its `port.txt`).
- **Orchestration:** spawn handler server.py:50241-50426 (registry:
  `spawned-pids.json`, `spawn_id = str(pid)`); inject 52069 →
  `_inject_text_into_session` 36047 fall-through; ask 52213 →
  `ask_session_and_wait` 36981; report_to = prompt footer
  `_wrap_prompt_with_return_address` 31156 hardcoding local loopback —
  cross-node needs a node-qualified return address. `_RETURN_ADDRESS_RE`
  (31035) disallows `:` — global refs must be parsed before normalization.
- **Group chats:** `~/.claude/group-chats/<slug>-<ts>.md` + sidecar `.json`
  with stable `uuid`; nudges deliver via `_inject_text_into_session`; all
  local-path bound (resolution, sandbox check server.py:34903, watcher
  34532). Cross-node = `host_node` in sidecar + global participant refs +
  proxied read/post/nudge.
- **Repo ops:** `_git()` 6491 canonical list-form runner; worktrees 41213;
  dirty 41111; PRs `_open_prs_cached` 41136 (no head SHA — add);
  `_get_pr_state` 3212; unpublished `_count_unpushed_commits` 9966; Vercel
  `vercel_deploy_status` 37785 (SHA-matched). Ship pipeline persisted-job
  pattern: `ship-jobs.json` atomic tmp+replace, log capped (38009) — model
  for fleet executor jobs. Repo identity today keyed by path everywhere;
  `_github_owner_repo_from_url` 6706 is github-only → federation.py
  `parse_remote_url` generalizes.
- **Transcript store:** `~/.claude/projects/<slug>/<sid>.jsonl`; slug =
  `re.sub(r"[^A-Za-z0-9]", "-", cwd)` (`_encode_project_slug` 2266) — the
  handoff path rewrite must re-encode the destination cwd and rewrite
  path-bearing fields. CCC sidecars (names/overrides/links) live under
  command-center state, not next to transcripts.
- **Hooks/provenance:** hooks write `live-state/<sid>.json` +
  `<sid>_writes` markers with tool + file fields — the natural feed for the
  provenance index; effective-cwd inference `_infer_effective_repo` 40941.
